\# Skills Style Guide



\## Every skill must include



\- Clear name

\- Clear description

\- Purpose

\- When to use

\- When not to use

\- Inputs

\- Outputs

\- Workflow

\- Best practices

\- Common mistakes

\- Related skills



\## Writing Rules



\- Be practical.

\- Be production-focused.

\- Prefer step-by-step workflows.

\- Include security reminders.

\- Include deployment reminders when relevant.

\- Never include secrets, tokens, passwords, or API keys.



\## File Structure



Each skill should use:
