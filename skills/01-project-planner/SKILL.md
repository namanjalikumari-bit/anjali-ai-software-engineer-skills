\# Anjali Project Planner Skill



\## Purpose



Plan a complete business software project before coding starts.



This skill is used when the user wants to build a website, dashboard, SaaS app, MERN project, admin panel, business platform, or client project.



\## When to Use



Use this skill when the user says:



\- Build a website

\- Make a business project

\- Create a full-stack app

\- Plan a MERN project

\- Build admin dashboard

\- Make SaaS MVP

\- Create doctor appointment website

\- Create PG management system

\- Create donation platform

\- Create hospital management system

\- Create e-commerce project

\- Start project from scratch



\## Main Goal



Convert a rough idea into a clear software development plan.



\## Always Ask or Infer



\- Business type

\- Target users

\- Admin users

\- Main features

\- Login/signup needed or not

\- Payment needed or not

\- File upload needed or not

\- Email/notification needed or not

\- Database type

\- Deployment target

\- Timeline

\- MVP features



\## Output Format



Always give output in this format:



1\. Project Overview

2\. Business Goal

3\. Target Users

4\. User Roles

5\. MVP Features

6\. Advanced Features

7\. Pages Required

8\. Admin Dashboard Modules

9\. Database Entities

10\. API Modules

\## Planning Workflow



\### Step 1: Understand Business



Identify:

\- What problem this project solves

\- Who will use it

\- Who will manage it

\- What data will be stored

\- What actions users can perform



\### Step 2: Define User Roles



Common roles:



\- Public visitor

\- Registered user

\- Staff

\- Admin

\- Super admin



\### Step 3: Define MVP



MVP means minimum features needed to launch.



Always separate:



\- Must-have features

\- Good-to-have features

\- Future features



\### Step 4: Define Pages



Common pages:



\- Home

\- About

\- Services

\- Contact

\- Login

\- Register

\- Dashboard

\- Admin Dashboard

\- Privacy Policy

\- Terms



\### Step 5: Define Database



For every project, suggest main entities.



Example for Doctor Appointment:



\- User

\- Doctor

\- Appointment

\- Payment

\- Report

\- Review

\- ContactMessage



\### Step 6: Define APIs



Example:



\- Auth APIs

\- User APIs

\- Admin APIs

\- Appointment APIs

\- Payment APIs

\- Upload APIs

\- Report APIs



\### Step 7: Define Deployment



Frontend:

\- Vercel



Backend:

\- Render



Database:

\- MongoDB Atlas or Supabase



Version Control:

\- GitHub



\## Best Practices



\- Do not start coding before planning.

\- Always include admin dashboard for business projects.

\- Always include database planning.

\- Always include deployment planning.

\- Always include security checklist.

\- Always include testing checklist.

\- Keep MVP simple.

\- Avoid overengineering in the first version.



\## Common Mistakes



Avoid:



\- No clear feature list

\- No admin dashboard

\- No database design

\- No API planning

\- No deployment plan

\- No `.env` planning

\- No README

\- No testing checklist

\- No role-based access planning
