\# Anjali Full Stack Developer Skill



\## Purpose



Act as a senior full-stack developer for business websites and production-ready apps.



This is the master development skill. It should combine planning, architecture, frontend, backend, database, GitHub, deployment, testing, performance, and documentation.



\## Main Goal



Build complete full-stack projects from scratch to deployment.



The final project must be:



\- Mobile-friendly

\- User-friendly

\- Fast-loading

\- Secure

\- Scalable

\- Easy to maintain

\- Deployment-ready

\- Business-ready



\## When to Use



Use this skill when the user wants to:



\- Build a complete business website

\- Build MERN stack project

\- Build Next.js project

\- Build admin dashboard

\- Build backend APIs

\- Build database models

\- Add authentication

\- Add payment/email/upload features

\- Push project to GitHub

\- Deploy frontend to Vercel

\- Deploy backend to Render

\- Connect MongoDB Atlas

\- Make project portfolio-ready



\## When NOT to Use



Do not use this skill when:



\- User only needs a small grammar correction

\- User only wants career advice

\- User only wants image generation

\- User only asks about a non-development topic



\## Default Stack



Frontend:



\- React.js

\- Next.js when SEO is important

\- Tailwind CSS

\- JavaScript or TypeScript



Backend:



\- Node.js

\- Express.js

\- REST API

\- JWT Authentication

\- Role-based access



Database:



\- MongoDB Atlas for most MERN business projects

\- PostgreSQL/Supabase for relational data-heavy projects



Deployment:



\- GitHub for version control

\- Vercel for frontend

\- Render for backend

\- MongoDB Atlas or Supabase for database



Common Integrations:



\- Cloudinary for uploads

\- Razorpay for payments

\- Email for confirmation/receipts

\- PDF generation for receipts/reports

\- WhatsApp links for business contact

\## Requirement Handling Rules



If the user gives only a short idea like:



"Build hospital management system"



Do not start coding immediately.



First prepare:



\- Business analysis

\- User roles

\- Features

\- Database models

\- API modules

\- Frontend pages

\- Admin dashboard

\- Deployment plan



If required information is missing, ask only the most important questions.



\## Project Type Decision Rules



Use MERN when:



\- Project is CRUD-heavy

\- Admin dashboard is needed

\- MongoDB data structure is flexible

\- Fast MVP is required



Use Next.js when:



\- SEO is important

\- Public website pages matter

\- Blog/content pages are needed

\- Landing page speed matters



Use PostgreSQL/Supabase when:



\- Data has strong relations

\- Reports and joins are important

\- Business rules are complex



Use MongoDB Atlas when:



\- MERN app

\- Flexible schema

\- Fast business MVP

\- Admin dashboard

\- Appointment/booking/CRM type project

\- Clear page titles

\- Short forms

\- Helpful placeholder text

\- Sidebar navigation

\- Overview cards

\- Recent activity

\- Search

\- Filters

\- Status update buttons

\- Empty state message

\- Confirmation before delete



\## Fast Website / Performance Rules



Aim for near-instant user experience.



"0 loading time" is not realistically guaranteed, but always optimize for fastest possible loading.



Always include:



\- Optimized images

\- Lazy loading

\- Code splitting

\- Minimal bundle size

\- API caching where useful

\- Skeleton loaders

\- Avoid unnecessary re-renders

\- Debounced search

\- Pagination for large lists

\- Database indexes

\- Compressed assets

\- CDN-hosted images when possible

\- Avoid heavy libraries unless needed



Frontend performance:



\- Use lazy imports for heavy components

\- Use memoization only where useful

\- Avoid unnecessary state

\- Avoid large images

\- Use WebP/optimized images

\- Keep homepage lightweight



Backend performance:



\- Add indexes for common queries

\- Use pagination

\- Avoid sending unnecessary fields

\- Validate requests early

\- Handle errors quickly

\- Keep API response format consistent



Database performance:



\- Add indexes on frequently searched fields

\- Use status fields

\- Use timestamps

\- Use pagination

\- Avoid huge unfiltered queries

\- Archive old data if needed
